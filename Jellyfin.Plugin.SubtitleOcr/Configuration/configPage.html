Path to a Subtitle Edit-compatible .nocr file. Leave empty to use the bundled Latin database. Databases trained in Subtitle Edit's nOCR window drop in directly.
Optional overrides matched on the subtitle's language (ISO 639 code, e.g. rus, ell). The path may be absolute or a bare file name resolved against the plugin's nocr data folder. Simplest option: drop a file named {language}.nocr (e.g. rus.nocr) into that folder and it is used automatically with no entry here. Resolution order per language: an entry below, then a drop-in {language}.nocr, then the database path above, then the bundled Latin database.
Fold dot runs ("...", ". . .", "..") into a single ellipsis character.
Fold an accented Latin letter that is foreign to the track's language to its base letter (a misread "Nothińg" to "Nothing"), keeping the accents that language legitimately uses. A known cast or character name keeps its accent.
Tag a track whose text describes sound ("[door slams]") as SDH when the source flags nothing, which is the norm for a remuxed disc.
Comma-separated ISO 639 codes (e.g. eng, und). Leave blank to extract every image-based subtitle language. Use und for untagged tracks.
Absolute floor for the word-space gap. The effective threshold is the larger of this and the factor below times the subtitle line height.
Scales the word-space threshold with subtitle resolution. 0.30 suits most discs; lower it if words run together, raise it if single words split apart.
If more than this fraction of a track's events are dropped, no SRT is written and any stale one is deleted. The track OCR'd too poorly (usually a wrong or untrained database).
SRT is universal; ASS adds font/size control and keeps a subtitle's top or mid-screen placement (via an alignment bucket). Auto writes ASS per track only when a cue is positioned away from the bottom, otherwise SRT.
Inserted into generated file names ({name}.{lang}.{tag}.srt) and shown as the subtitle title in Jellyfin, marking the file as plugin-created. The plugin only overwrites or deletes files carrying this tag, so it never touches source or hand-made subtitles. Blank writes {name}.{lang}.srt and gives up that protection.
Template with {code} (ISO 639-1); .dic and .aff are appended. Defaults to the wooorm/dictionaries collection. Downloaded dictionaries keep their own (often GPL/LGPL) licenses.
Template with {code} (ISO 639-2/T, e.g. eng, deu). Defaults to Subtitle Edit's MIT-licensed replace lists. A missing list is skipped.
Re-download a cached dictionary or fix list once it is older than this many days, picking up upstream changes. 0 never refreshes (delete a file in the dictionaries folder to force it).
An unchanged file already probed within this many days is skipped. A changed file is always re-probed. Set to 0 to re-probe only files that changed.
